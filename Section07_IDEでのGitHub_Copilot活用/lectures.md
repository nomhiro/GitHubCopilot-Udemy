## レクチャー2: コード補完機能の使い方
**時間目安: 12分**

### 学習目標
- GitHub Copilotでサポートされている言語を把握する
- 自動提案機能の仕組みと使い方を理解する
- コーディングコメントを活用した提案の質向上方法を習得する

### 内容

#### GitHub Copilotでサポートされている言語

GitHub Copilotは、以下のプログラミング言語とフレームワークに対して特に優れたサポートを提供します：

| 言語 | 特徴 |
|------|------|
| Python | データサイエンス、Web開発、自動化 |
| JavaScript | フロントエンド、Node.js開発 |
| Java | エンタープライズアプリケーション |
| TypeScript | 型安全なJavaScript開発 |
| Ruby | Web開発（Rails等） |
| Go | システムプログラミング、マイクロサービス |
| C# | .NET開発、ゲーム開発 |
| C++ | システムプログラミング、パフォーマンス重視のアプリ |

> **ヒント**: これらの言語以外でもGitHub Copilotは支援を提供できますが、上記の言語では特に高精度な提案が得られます。

#### 自動提案機能

Copilotは、ユーザーが入力している間にリアルタイムでコード提案を行います：

- **現在の行の補完**: 入力中の行を完成させる提案
- **新しいコードブロック**: まったく新しいコードブロックの提案
- **ゴーストテキスト**: 提案はグレー表示のテキストとして表示される

**提案の操作方法**:
- すべてを受け入れる
- 一部のみ受け入れる
- 提案を無視する

この機能により、以下の作業が軽減されます：
- 構文の検索
- ロジックのトラブルシューティング
- 一般的なパターンの繰り返し記述

#### コーディングスタイルへの適応

GitHub Copilotは、ユーザーのコーディングスタイルを学習し、適応します：

| 適応機能 | 説明 |
|----------|------|
| **メソッドの実装** | メソッド名の入力を開始すると、確立したコーディングスタイルに従って実装全体を提案 |
| **名前付け規則** | 変数、関数、クラスに対する好みの命名規則を学習 |
| **書式設定** | インデントスタイル、ブラケットの配置、その他の書式設定の好みに適応 |
| **コメントスタイル** | インラインコメント、ブロックコメント、ドキュメント文字列のスタイルを模倣 |
| **設計パターン** | プロジェクトで一貫して使用されている設計パターンに合ったコードを提案 |

#### コーディングコメントを利用した提案

GitHub Copilotはコードベース内のコメントを活用して、より適切な提案を生成します。

**コメントコンテキストの理解**:
- **自然言語処理（NLP）**: コメントの背後にある意味と意図を解釈
- **コンテキスト分析**: 周囲のコードとの関連性を分析

**利用されるコメントの種類**:

| コメントタイプ | 説明 | 例 |
|----------------|------|-----|
| インラインコメント | 特定のコード行の横にある簡単な説明 | `// ユーザー名を取得` |
| ブロックコメント | 関数またはクラスを記述する長い説明 | `/* 関数の詳細説明 */` |
| Docstring | 正式なドキュメント文字列（Python等） | `"""関数の説明"""` |
| TODOコメント | 今後の実装や改善に関するメモ | `// TODO: エラー処理を追加` |
| APIドキュメント | 関数の使用方法とパラメーターの説明 | JSDoc、XMLDoc等 |

#### コメント駆動コード生成

**1. 関数の実装**

コメントで関数の説明を書くと、Copilotが実装全体を提案します：

```python
# 文字列を反転させる関数
def reverse_string(s):
    return s[::-1]  # Copilotが提案
```

**2. 変数の名前付け**

コメントは変数名の提案にも影響します：

```python
# ユーザーのお気に入りの書籍リスト
favorite_books = []  # Copilotが提案した変数名
```

**3. アルゴリズムの選択**

特定のアルゴリズムをコメントで説明すると、その方法に沿ったコードを提案：

```python
# バブルソートアルゴリズムを使用してリストをソート
# 1. 隣接する要素を比較
# 2. 順序が間違っていれば交換
# 3. リスト全体を繰り返し処理
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```

#### ベストプラクティス

1. **明確なコメントを書く**: やりたいことを具体的に説明
2. **意味のある変数名を使用**: コンテキストを伝える名前を選ぶ
3. **段階的にアプローチ**: 複雑な処理は小さなステップに分ける
4. **既存のコーディング規約を維持**: プロジェクトの一貫性を保つ

### 内容

#### 複数提案ペイン

GitHub Copilotはコードブロックで作業している際に、グレー表示のコードスニペット（ゴーストテキスト）として提案を表示します。

**コントロールパネルの表示方法**:
1. 提案が表示されたらカーソルを合わせる
2. GitHub Copilotコントロールパネルが表示される
3. 前方または後方矢印ボタンをクリックして提案を切り替え

この機能により、同じ問題に対する複数のアプローチをすばやく評価でき、特定のコンテキストに最適なソリューションを選択できます。

#### キーボードショートカット

**提案の切り替え**:

| 操作 | macOS | Windows/Linux |
|------|-------|---------------|
| 次の提案を表示 | `Option (⌥) + ]` または `Alt + ]` | `Alt + ]` |
| 前の提案を表示 | `Option (⌥) + [` または `Alt + [` | `Alt + [` |

**提案の操作**:

| 操作 | キーボードショートカット |
|------|------------------------|
| 提案を受け入れる | `Tab` |
| 提案を拒否する | `Esc` |
| 単語単位で受け入れる | `Ctrl + →` (Windows) / `Cmd + →` (Mac) |

#### 複数提案のメリット

- **迅速な反復**: コーディングフローを中断することなくアプローチを比較
- **最適な選択**: ゼロから始める代わりに、数秒以内にさまざまな実装を評価
- **学習機会**: 異なる実装パターンを見ることでスキル向上

## レクチャー8: GitHub MCP ServerによるAIワークフローの強化
**時間目安: 10分**

### 学習目標
- モデルコンテキストプロトコル（MCP）の概念を理解する
- GitHub MCP Serverの利点と動作を把握する
- MCPを活用したAIワークフローの効率化方法を学ぶ

### MCPとは

MCP（モデルコンテキストプロトコル）は、**AIツールのUSB-C標準**のようなものです。AIモデルが必要なツールやデータソースに接続するための一貫した安全な方法を提供します。

#### MCPの主な特徴

| 特徴 | 説明 |
|------|------|
| **ツールライブラリへのアクセス** | AIモデルがすぐに使用できるツールのライブラリにアクセス可能 |
| **柔軟性** | ワークフローの一貫性を維持しながら、さまざまなAIプロバイダーと連携可能 |
| **統合性** | 既存の開発環境とプロセスへの統合が容易 |

### MCPアーキテクチャ

MCPクライアント（Claude、IDE、または別のツール）がMCPサーバーと接続するサービスと対話できる3つの一般的な方法があります。

#### 1. ローカルデータとのローカル通信（トップパス）

MCPクライアントは、コンピューター上で実行されているMCPサーバーと直接通信します。そのサーバーはローカルデータソース（ファイル、データベース、コンピューターに格納されているその他のリソース）に接続します。

**使用するタイミング**: ローカル開発や、コンピューター上でプライベートなままのデータにすばやくアクセスする場合

#### 2. リモートサービスへのブリッジ（中間パス）

MCPクライアントはローカルで実行されているMCPサーバーに接続しますが、このサーバーはWeb APIを呼び出してインターネット上のリモートサービスにブリッジします。

**使用するタイミング**: ローカルツールがリモートサービスから情報をフェッチまたは更新する必要があり、キャッシュ、セキュリティチェック、データの前処理などを処理するためにローカルサーバーを間に入れるとメリットがある場合

#### 3. インターネット経由のリモート通信（下部パス）

MCPクライアントは、インターネット上に完全に存在するMCPサーバーに接続します。そのリモートサーバーは、Web APIを介して他の外部サービスと通信します。

**使用するタイミング**: 必要なリソースまたは計算をローカルで実行できない場合（クラウドベースのコンピューティング、SaaSプラットフォーム、オンラインでのみ存在するサードパーティの統合を使用する場合など）

### GitHub MCP Serverを使用する理由

通常、ローカルMCPサーバーを使用するには、Docker、トークン管理、手動構成が必要です。これにより、セットアップが遅くなり、GitHub.comなどのWebクライアントとの統合がブロックされる可能性があります。

GitHub MCP Serverはこれらの問題を解決します：

| 利点 | 説明 |
|------|------|
| **Docker不要** | Docker または手動の構成ファイルが不要 |
| **簡単な認証** | 高速認証のためのワンクリック OAuth ログイン |
| **マルチプラットフォーム対応** | Web、デスクトップ、モバイル環境間でシームレスに作業可能 |
| **エンタープライズサポート** | Entra や Auth0 などのエンタープライズ ID プロバイダーをサポート |
| **自動スケーリング** | 使用状況のニーズに合わせて自動的にスケーリング |

### GitHub MCP Serverの動作

GitHub MCP Serverは、GitHub Copilotやその他のAIツールをリポジトリに直接接続するオープンソースサーバーです。

#### 主な機能

- **コード分析と要約**: プロジェクトをより深く理解するためのコード分析
- **Issue管理**: 問題を作成して管理
- **Pull Request支援**: Pull Requestの作成と管理
- **タスク自動化**: リポジトリのトリアージとタスク追跡を自動化

#### 利用可能なツール

現在、GitHub MCP Serverには**30を超えるツール**が用意されています：

| カテゴリ | 機能 |
|---------|------|
| **ファイル操作** | 問題の追加、ファイルの編集、ブランチの作成 |
| **優先度管理** | Pull Requestと問題のランク付け |
| **セマンティック検索** | コードベース全体のセマンティック検索 |
| **自動化** | ワークフローの自動化と効率化 |

### IDEでのMCP活用

IDEでGitHub MCP Serverを活用することで、以下のワークフローが効率化されます：

1. **問題トリアージの自動化**
   - Issue の優先度を自動的に評価
   - 関連するコードファイルを特定
   - 解決策の提案を生成

2. **コードレビューの強化**
   - Pull Request の内容を分析
   - 潜在的な問題を特定
   - 改善提案を自動生成

3. **プロジェクト理解の促進**
   - コードベース全体のセマンティック検索
   - プロジェクト構造の可視化
   - ドキュメントの自動生成

### VS CodeでのGitHub MCPサーバー設定

GitHub MCP Serverの概念を理解したところで、VS Codeでセットアップして実際に使用する方法を見てみましょう。

#### OAuthを使用した設定（推奨）

最も簡単な方法は、OAuthを使用した設定です。複雑なセットアップなしで、AIワークフローをコーディング環境に直接統合できます。

**設定手順**:

1. **コマンドパレットを開く**
   - Windows/Linux: `Ctrl + Shift + P`
   - Mac: `Cmd + Shift + P`

2. **MCPサーバーを追加**
   - 「MCP: サーバーの追加」と入力し、Enterキーを押す

3. **接続タイプを選択**
   - 一覧から「**HTTP (HTTP または Server-Sent イベント)**」を選択

4. **サーバーURLを入力**
   - サーバーURLフィールドに以下を入力:
   ```
   https://api.githubcopilot.com/mcp/
   ```

5. **サーバーIDを設定**
   - Enterキーを押してデフォルト値を使用、または任意のカスタムIDを入力

6. **構成の保存場所を選択**
   - **ユーザー設定**: すべてのプロジェクトで使用可能
   - **ワークスペース設定**: 現在のプロジェクトのみで使用

7. **OAuth認証**
   - プロンプトが表示されたら「**許可**」を選択してGitHubアカウントにサインイン

> **ヒント**: セットアップ完了後、GitHub MCP Serverはすぐにプロジェクトで使用できます。タスクの自動化、問題の管理、コードの分析をエディター内で直接開始できます。

#### 個人用アクセストークン（PAT）を使用した設定

高度な制御が必要な場合は、個人用アクセストークン（PAT）を使用できます。

**手順**:

1. **PATを作成**
   - GitHubアカウントで、リポジトリとパッケージスコープを含むPATを作成

2. **OAuth設定の代わりにPATを使用**
   - 上記のOAuth設定手順に従い、OAuth認証をスキップ

3. **構成ファイルにヘッダーを追加**
   ```json
   "headers": {
     "Authorization": "Bearer ${input:github_token}"
   }
   ```

4. **入力プロンプトを追加**
   ```json
   "inputs": [
     {
       "id": "github_token",
       "type": "promptString",
       "description": "GitHub Personal Access Token",
       "password": true
     }
   ]
   ```

5. **MCPサーバーを再起動**
   - VS CodeでMCPサーバーを再起動し、プロンプトが表示されたらPATを入力

#### Dockerを使用したローカルセットアップ（オプション）

GitHub Enterprise Serverを使用している企業向けのオプションです。

> **注意**: PAT制限付きのGitHub Enterprise Serverを使用している場合は、組織のポリシーで許可されているAPIスコープにのみアクセスできます。すべてのエンドポイントが制限されている場合、MCPサーバーは使用できません。

**前提条件**:
- Dockerがインストールされ、実行されていること
- 必要なスコープを持つPAT

**構成例**:
```json
{
  "inputs": [
    {
      "type": "promptString",
      "id": "github_token",
      "description": "GitHub Personal Access Token",
      "password": true
    }
  ],
  "servers": {
    "github": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${input:github_token}"
      }
    }
  }
}
```

#### トラブルシューティング

GitHub MCPサーバーの使用中に問題が発生した場合のチェックリスト:

| 確認項目 | 対処法 |
|---------|--------|
| **GitHubアカウント** | VS Code内でGitHubアカウントにサインインしていることを確認 |
| **PAT設定** | PATを使用している場合、正しいスコープを持ち、正しく入力されているか確認 |
| **構成ファイル** | 入力ミスやフィールドの欠落がないか再確認 |
| **Docker** | Dockerを使用している場合、インストールされ、アクティブに実行されているか確認 |
| **接続問題** | 一時的な接続の問題を解決するには、VS CodeまたはMCPサーバーを再起動 |

### まとめ

- MCPはAIツールを接続するための標準プロトコル
