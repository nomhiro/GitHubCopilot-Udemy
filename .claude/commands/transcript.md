# AI音声用トランスクリプト生成

## 対象セクション

$ARGUMENTS

## 概要

Udemy動画講座でAI音声を生成するためのトランスクリプトを作成します。
スライド（slide.md）をベースに、追加情報で補足しながら各スライドごとにトランスクリプトを作成してください。

## 出力ファイル

`フォルダ名_transcript.md` に出力します。
例: Section05_GitHub_Copilot_Chat → Section05_GitHub_Copilot_Chat_transcript.md

## 禁止・置換ルール

### 禁止表現
- 「なのです」という表現は使わない
- 誇張した表現は避ける
- 括弧で補足しない（トランスクリプトはそのまま読み上げます）
- 「以下の」「以下のような」という表現は使わない

### 置換ルール
- 「＆」→「と」に置き換える
- 「第一に、第二に、第三に...」→「一つ目は、二つ目は、三つ目は、四つ目は、最後に」に置き換える

## 話し方のトーン

### スライド間の自然な繋がり
スライドの切り替え時に、自然な接続表現を入れてください。

- 「それでは〜を見ていきましょう」
- 「さて、次は〜についてです」
- 「続いて〜について説明します」
- 「ここからは〜に進みます」

### 聞き手への語りかけ
時々、聞き手に語りかける表現を入れて親しみやすくしてください。

- 「〜ですよね」
- 「〜と困りますよね」
- 「〜という経験はありませんか」
- 「〜は便利ですよね」

### 話の展開を明確にする
トピックの切り替えを明確にする表現を入れてください。

- 「一般的な〜の説明をしたところで、具体的に〜を見ていきましょう」
- 「概念を理解したところで、実際の使い方を見ていきます」
- 「基本を押さえたので、応用的な内容に進みましょう」

## 図解・アーキテクチャ図の説明

図を説明する際は、「〜図を見ると」ではなく、具体的な位置を示して簡潔に説明してください。

### 良い例
- 「図の右下の黄色い部分です」
- 「画面上部のメニューバーにある」
- 「フローの3番目のステップを見てください」

### 悪い例
- 「この図を見ると〜ということがわかります」
- 「以下の図をご覧ください」

## トランスクリプトの構成

各スライドに対して、見出しとトランスクリプト本文を記載してください。

```markdown
## スライド1: スライドタイトル

トランスクリプト本文...

## スライド2: スライドタイトル

トランスクリプト本文...
```

## 参考情報の取得

必要に応じて以下のツールを使用してください：
- **Microsoft Docs MCP**: Microsoft Learn から最新情報を取得
- **Context7 MCP**: ライブラリのドキュメント取得
- **DeepWiki MCP**: GitHubリポジトリの情報取得

## 実行

$ARGUMENTS フォルダ内の slide.md を参照し、トランスクリプトファイルを作成してください。
初心者にわかりやすく、聞きやすいトランスクリプトを心がけてください。
