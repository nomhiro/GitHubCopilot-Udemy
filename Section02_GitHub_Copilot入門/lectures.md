## Lecture 1: GitHub Copilotとは

### 学習目標
- GitHub Copilotの基本概念を理解する
- AIペアプログラマとしての役割を説明できる
- GitHub Copilotの効果と主要機能を把握する

### 内容

#### GitHub Copilotの概要
- **定義**: OpenAIと共同開発されたAIペアプログラマサービス
- **技術基盤**: OpenAI Codexシステム（GPT-3より優れたコード生成能力）
- **機能**: オートコンプリート形式でリアルタイムにコード候補を提案
- **対応言語**: Python、JavaScript、TypeScript、Ruby、Go、C#、C++など多数

#### 対応IDE
- Visual Studio Code
- Visual Studio
- Vim / Neovim
- JetBrains スイート（IntelliJ、PyCharm等）

#### GitHub Copilotの効果（調査データ）

| 指標 | 数値 |
|------|------|
| AIによって記述された新しいコードの割合 | **46%** |
| 開発者の生産性向上 | **55%** |
| 満足のいく作業に集中できると感じた開発者 | **74%** |

#### GitHub Copilotの主要機能

1. **コード補完**: インラインでリアルタイムにコード提案
2. **チャット用Copilot**: ChatGPTライクなインターフェイスでコード分析・テスト生成
3. **Pull Request用Copilot**: PR説明の自動生成（GPT-4モデル使用）
4. **CLI用Copilot**: ターミナルでのコマンド構成支援

#### AIペアプログラマとは
- 開発者と協働してコーディングを支援
- コンテキストを理解して適切な提案を生成
- 単なるオートコンプリートではなく、複数行のコード提案が可能

## Lecture 2: GitHub Copilotの主な機能

### 学習目標
- GitHub Copilotの4つの主要機能を理解する
- 各機能の概要と用途を把握する

### 内容

#### 1. コード補完機能

コードを入力している間、Copilotがリアルタイムで提案を行います。

**特徴**:
- 入力中に淡色のテキスト（ゴーストテキスト）として提案が表示される
- 現在の行の補完から、新しいコードブロック全体の提案まで対応
- 提案を受け入れるか、無視するかを選択できる

**例示**: コメントからコードを生成
```python
# 文字列を反転させる関数
def reverse_string(s):
    return s[::-1]  # Copilotが自動で提案
```

#### 2. Copilot Chat機能

自然言語で対話しながらコーディングの支援を受けられます。

**特徴**:
- ChatGPTのような対話形式でコードについて質問できる
- コードの説明、デバッグ支援、新しいコードの生成が可能
- 新しい概念の学習や構文の確認にも活用できる

**例示**: 質問してコードを生成
```
質問: 「Pythonでファイルを読み込む方法を教えて」
→ Copilotがサンプルコードと説明を提供
```

#### 3. インラインチャット機能

エディター内で直接Copilotと対話できます。

**特徴**:
- コードを選択して、その場で説明や修正を依頼できる
- 画面を切り替えずにコーディングを継続できる
- コンテキストに基づいた的確な回答を得られる

**例示**: コードを選択して説明を求める
```
コードを選択 → 「このコードを説明して」と依頼
→ 選択したコードの動作を日本語で解説
```

#### 4. CLI用Copilot

ターミナル/コマンドラインでもCopilotの支援を受けられます。

**特徴**:
- シェルコマンドの構成を支援
- コマンドの説明や代替案を提案
- 複雑なコマンドラインタスクを効率化

**例示**: コマンドの説明を求める
```
$ gh copilot explain "find . -name '*.txt' -mtime +30"
→ 30日以上前に変更されたテキストファイルを検索するコマンドの説明
```

> **ヒント**: 各機能の詳細な操作方法は、Section07「IDEでのGitHub Copilot活用」で学習します。

## Lecture 3: GitHub Copilotの効果的な活用ポイント

### 学習目標
- Copilotから良い提案を得るためのポイントを理解する
- 実践的な活用シーンを把握する
- Copilotの学習特性を理解する

### 内容

#### 効果的な活用のための3つのポイント

| ポイント | 説明 | 例 |
|---------|------|-----|
| **1. 明確なコンテキスト提供** | 変数名や関数名を意味のあるものにする | `x`ではなく`user_age`、`f`ではなく`calculate_tax` |
| **2. コメントの活用** | やりたいことを自然言語で説明する | `# ユーザーの年齢をバリデートする関数` |
| **3. 段階的なアプローチ** | 複雑な処理は小さなステップに分ける | 大きな関数を複数の小さな関数に分割 |

#### 実践的な活用シーン

**1. 新しいコードの作成**
- コメントで意図を説明 → Copilotが実装を提案
- 関数名を入力 → 関数全体の実装を提案

**2. 既存コードの理解と説明**
- 知らないコードを選択 → 説明を依頼
- 複雑なロジックの動作を確認

**3. テストコードの生成**
- 関数を選択 → テストケースを自動生成
- エッジケースも含めた包括的なテスト提案

```python
def add(a, b):
    return a + b

# Copilotが生成するテスト例
def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0, 0) == 0
```

**4. リファクタリング支援**
- コードを選択 → より効率的な書き方を提案
- 重複コードの共通化を支援

#### Copilotの学習特性

- **コンテキスト理解**: 開いているファイルや周囲のコードから学習
- **スタイル適応**: プロジェクトの命名規則やコーディングスタイルに適応
- **継続的な改善**: 相互作用を重ねるほど的確な提案が得られる

> **重要**: Copilotの提案は必ずレビューしてから採用してください。提案が常に正しいとは限りません。

## Lecture 4: 設定と構成

### 学習目標
- GitHub Copilotの設定方法を理解する
- VS Codeでの拡張機能インストール手順を習得する
- 組織での管理方法を把握する

### 内容

#### GitHub Copilotへのサインアップ
1. GitHubのプロフィール写真を選択 → **設定**
2. 左メニューの「コード、計画、自動化」から **Copilot** を選択
3. 無料試用版またはサブスクリプションを設定

#### VS Codeでの拡張機能インストール
1. Visual Studio Marketplaceで「GitHub Copilot」を検索
2. **インストール** を選択
3. VS Codeが開いたら拡張機能タブで **インストール** を選択
4. 初回はGitHubにサインインが必要

#### 有効/無効の切り替え
- VS Code下部の状態アイコンをクリック
- **有効にする** または **無効にする** を選択
- グローバルまたは特定の言語に対して無効化可能

#### インライン候補の設定
1. **ファイル** → **ユーザー設定** → **設定**
2. **拡張機能** → **GitHub Copilot** を選択
3. 「エディター: オートコンプリートを有効にする」で制御

#### 組織での管理
- Organization/Enterpriseでのポリシー設定
- ユーザーへのライセンス割り当て
- コンテンツ除外の設定

## Lecture 5: トラブルシューティング

### 学習目標
- 一般的な問題の解決方法を理解する
- ログファイルと診断情報の確認方法を習得する

### 内容

#### よくある問題と解決方法

| 問題 | 解決方法 |
|------|----------|
| 提案が表示されない | 拡張機能の有効化を確認、ネットワーク接続を確認 |
| 認証エラー | GitHubアカウントの再認証 |
| 遅延が発生 | ネットワーク状況の確認、プロキシ設定の見直し |

#### 一般的な問題の原因
- ネットワーク制限
- ファイアウォール設定
- プロキシ接続の問題

#### ログファイルの確認
コマンドパレット（`Ctrl+Shift+P` / `Cmd+Shift+P`）で以下を実行：
- **「開発者: ログ ファイルを開く」**
- **「開発者: 拡張機能のログ フォルダーを開く」**

#### Electronログの表示
- **ヘルプ** → **開発者ツールの切り替え**

#### 診断情報の収集
1. コマンドパレットを開く
2. 「Diagnostics」と入力
3. **「GitHub Copilot: 診断情報の収集」** を選択

#### ステータスの確認
- VS Code下部のGitHub Copilotステータスアイコンを確認
- アイコンの色や状態で接続状況を判断
