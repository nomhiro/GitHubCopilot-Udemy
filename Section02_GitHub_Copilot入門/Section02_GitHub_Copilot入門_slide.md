# GitHub Copilotの概要

## GitHub Copilotとは？

- **定義**: OpenAIと共同開発されたAIペアプログラマサービス
- **技術基盤**: OpenAI Codexシステム（GPT-3より優れたコード生成能力）
- **機能**: オートコンプリート形式でリアルタイムにコード候補を提案

## 対応プログラミング言語

Python、JavaScript、TypeScript、Ruby、Go、C#、C++ など多数

<span class="red">試験ポイント: 技術基盤は「OpenAI Codex」です</span>

# コード補完機能

## リアルタイムでコード提案

- 入力中に淡色のテキスト（ゴーストテキスト）として提案が表示される
- 現在の行の補完から、新しいコードブロック全体の提案まで対応
- 提案を受け入れるか、無視するかを選択できる

## 例示: コメントからコードを生成

```python
# 文字列を反転させる関数
def reverse_string(s):
    return s[::-1]  # Copilotが自動で提案
```

---

# Copilot Chat機能

## ChatGPTライクな対話型インターフェイス

- 自然言語で対話しながらコーディングの支援を受けられる
- コードの説明、デバッグ支援、新しいコードの生成が可能
- 新しい概念の学習や構文の確認にも活用できる

## 活用例

- 「Python でバイナリ検索を実装するには？」
- 「このコードの意味を説明して」
- 「このエラーの原因は？」

---

# インラインチャット機能

## エディター内で直接対話

- コードを選択して、その場で説明や修正を依頼できる
- 画面を切り替えずにコーディングを継続できる
- コンテキストに基づいた的確な回答を得られる

## 例示

```
コードを選択 → 「このコードを説明して」と依頼
→ 選択したコードの動作を日本語で解説
```

---

# CLI用Copilot

## ターミナルでもCopilotの支援

- シェルコマンドの構成を支援
- コマンドの説明や代替案を提案
- 複雑なコマンドラインタスクを効率化

## 例示

```bash
$ gh copilot explain "find . -name '*.txt' -mtime +30"
→ 30日以上前に変更されたテキストファイルを検索するコマンドの説明
```

# 効果的な活用ポイント

---

# 効果的な活用のための3つのポイント

## より良い提案を得るために

<span class="red">AIへの入力の質が重要です</span>

| ポイント | 説明 | 例 |
|----------|------|-----|
| **1. 明確なコンテキスト** | 変数名や関数名を意味のあるものにする | `x`ではなく`user_age` |
| **2. コメントの活用** | やりたいことを自然言語で説明する | `# ユーザーの年齢をバリデート` |
| **3. 段階的なアプローチ** | 複雑な処理は小さなステップに分ける | 大きな関数を分割 |

---

# 実践的な活用シーン

## Copilotが役立つ4つのシーン

| シーン | 説明 |
|--------|------|
| **新しいコードの作成** | コメントで意図を説明 → 実装を提案 |
| **既存コードの理解** | 知らないコードを選択 → 説明を依頼 |
| **テストコードの生成** | 関数を選択 → テストケースを自動生成 |
| **リファクタリング支援** | コードを選択 → より効率的な書き方を提案 |

---

# テストコード生成の例

## 関数からテストを自動生成

```python
def add(a, b):
    return a + b

# Copilotが生成するテスト例
def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0, 0) == 0
```

<span class="red">試験ポイント: 単体テスト生成は「/tests」コマンドで実行</span>

# Copilotの学習特性

## 理解しておくべきポイント

- **コンテキスト理解**: 開いているファイルや周囲のコードから学習
- **スタイル適応**: プロジェクトの命名規則やコーディングスタイルに適応
- **継続的な改善**: 相互作用を重ねるほど的確な提案が得られる

<span class="red">試験ポイント: 出力の品質は「プロンプトをどの程度適切に作成したか」で決まります</span>

> **重要**: Copilotの提案は必ずレビューしてから採用してください

# 設定と構成

# GitHub Copilotへのサインアップ

## サインアップ手順

1. GitHubのプロフィール写真を選択 → **設定**
2. 左メニューの「コード、計画、自動化」から **Copilot** を選択
3. 無料試用版またはサブスクリプションを設定

## プラン

- **GitHub Copilot Free**: 月2,000コード補完、50チャット要求
- **GitHub Copilot Pro**: 無制限のコード補完とチャット
- **GitHub Copilot Business**: 組織向け管理機能付き
- **GitHub Copilot Enterprise**: 大規模組織向け高度機能

---

# VS Codeでの拡張機能インストール

## インストール手順

1. Visual Studio Marketplaceで「GitHub Copilot」を検索
2. **インストール** を選択
3. VS Codeが開いたら拡張機能タブで **インストール** を選択
4. 初回はGitHubにサインインが必要

<span class="red">試験ポイント: VS Code以外にも Visual Studio、JetBrains IDE、Neovim がサポートされています</span>

---

# 有効/無効の切り替え

## 操作方法

1. VS Code下部の状態アイコンをクリック
2. **有効にする** または **無効にする** を選択

## 無効化のオプション

- グローバルに無効化
- 特定の言語に対して無効化

## インライン候補の設定

**ファイル** → **ユーザー設定** → **設定** → **拡張機能** → **GitHub Copilot**

# よくある問題と解決方法

## 一般的なトラブル

| 問題 | 解決方法 |
|------|----------|
| 提案が表示されない | 拡張機能の有効化を確認、ネットワーク接続を確認 |
| 認証エラー | GitHubアカウントの再認証 |
| 遅延が発生 | ネットワーク状況の確認、プロキシ設定の見直し |

## 一般的な原因

- ネットワーク制限
- ファイアウォール設定
- プロキシ接続の問題

<span class="red">試験ポイント: 「言語がサポートされていない」は一般的な原因ではありません</span>

# ログファイルと診断情報

## ログファイルの確認

コマンドパレット（`Ctrl+Shift+P` / `Cmd+Shift+P`）で：
- **「開発者: ログ ファイルを開く」**
- **「開発者: 拡張機能のログ フォルダーを開く」**

## 診断情報の収集

1. コマンドパレットを開く
2. 「Diagnostics」と入力
3. **「GitHub Copilot: 診断情報の収集」** を選択
