# GitHub Copilotの4つのプラン

| プラン | 対象 | 主な特徴 |
|--------|------|----------|
| **Free** | 個人（入門） | 月2,000補完、50チャットの制限付き |
| **Pro** | 個人開発者 | 無制限の補完とチャット |
| **Business** | 組織・チーム | 管理機能、セキュリティ、IP補償 |
| **Enterprise** | 大規模企業 | Business + ナレッジベース |

# Free と Pro の機能比較

| 機能 | Free | Pro |
|------|:----:|:---:|
| コード補完 | 月2,000回 | **無制限** |
| チャットメッセージ | 月50回 | **無制限** |

| プライベート拡張機能の構築 | ✅ | ✅ |

<span class="red">試験ポイント: Freeの制限値（2,000/50）</span>

---

# Free/Proで利用できない機能

以下はBusiness/Enterpriseプラン限定

| 機能 | 説明 |
|------|------|
| ユーザー管理 | 組織メンバーへのライセンス管理 |
| トレーニング除外 | コードがAIトレーニングに使われない |
| IP補償 | 知的財産に関する法的保護 |
| コンテンツ除外 | 特定ファイルをCopilot分析から除外 |
| SAML SSO | 企業認証システムとの統合 |
| 使用状況メトリック | 利用状況の監視・分析 |

---

# パブリックコードフィルター

## すべてのプランで利用可能な機能

- GitHubの**公開リポジトリと一致するコード提案をフィルタリング**
- ライセンス問題を回避するための重要な機能
- オープンソースのコードをそのまま提案しないようにする

<span class="red">試験ポイント: パブリックコードフィルターは全プランで利用可能</span>

# ポリシー管理の重要性

## 組織でCopilotを使う際のメリット

### 1. 組織レベルでの一括管理
- チーム全体に一貫したポリシーを適用

### 2. コンプライアンス対応
- 組織のプライバシーポリシーに従った機密データの取り扱い

### 3. 監査対応
- 詳細な監査ログへのアクセス

# Copilot Enterpriseの機能とナレッジベース

---

# Copilot Enterprise専用機能

| 機能 | 説明 |
|------|------|
| **プライベートコードベース対応** | 組織固有のコードを理解した応答 |
| **ナレッジベース** | 組織のドキュメントに基づいた回答 |

## Enterprise限定のカスタマイズ

- 組織のコーディング規約に沿った提案
- 内部ドキュメントを参照した回答
- プライベートコードベースの理解

<span class="red">試験ポイント: ナレッジベースはEnterprise専用！</span>

---

# ナレッジベース機能とは

## 定義
組織固有の情報やドキュメントをCopilotに提供する仕組み

## メリット

| メリット | 内容 |
|----------|------|
| 規約準拠 | 組織のコーディング規約に沿った提案 |
| 内部参照 | 内部ドキュメントを参照した回答 |
| コード理解 | プライベートコードベースの理解 |

---

# GitHub Enterprise Cloudの要件

## Copilot Enterpriseの前提条件

| Copilotプラン | 必要なGitHubプラン |
|--------------|-------------------|
| Free/Pro | 個人アカウントでOK |
| Business | GitHub Team以上 |
| **Enterprise** | **GitHub Enterprise Cloud必須** |

## 2つのサブスクリプションが必要

Copilot Enterpriseを利用するには：

1. **GitHub Enterprise Cloud**（プラットフォーム側）
2. **GitHub Copilot Enterprise**（Copilot側）

# Business vs Enterprise：選択基準

| 選択基準 | Business | Enterprise |
|----------|----------|------------|
| **チーム規模** | 100人未満 | 100人以上の大規模組織 |
| **認証要件** | 基本的なSSO | SSO/SCIM必須 |
| **カスタマイズ** | 不要 | 組織固有のコード対応が必要 |
| **コンプライアンス** | 標準的 | 厳格な規制対応が必要 |
| **GitHub環境** | GitHub Team | GitHub Enterprise Cloud |

---

# プラン選択の考慮事項①

## 1. データのプライバシーとセキュリティ

- プランごとに異なるレベルのプライバシー対策
- **堅牢なプライバシー制御はBusiness/Enterprise限定**
- 機密データを扱う組織はBusiness以上を検討

## 2. ポリシー管理

- 組織レベルでのCopilotポリシー管理の必要性
- Business/Enterpriseで包括的なポリシー管理が可能

---

# プラン選択の考慮事項②

## 3. データの収集と保持

- データプライバシー規制への準拠
- 個人ユーザーはプロンプト・提案の収集/保持を選択可能

## 4. IP補償とデータプライバシー

- **Business/EnterpriseでIP補償を提供**
- 法律、セキュリティ、顧客の問題を回避
- 企業利用には必須の考慮事項

<span class="red">試験ポイント: IP補償はBusiness以上で提供</span>

# 試験対策：頻出ポイント

<span class="red">必ず覚えること：</span>

1. **Freeの制限値**: 月2,000コード補完、50チャットメッセージ
2. **パブリックコードフィルター**: 全プランで利用可能
3. **IP補償**: Business/Enterprise限定
4. **ナレッジベース**: Enterprise限定
5. **Enterprise要件**: GitHub Enterprise Cloud必須

# よくある誤答に注意！

| 誤答例 | 正解 |
|--------|------|
| ナレッジベースはBusinessで使える | ❌ Enterpriseのみ |
| IP補償はProで使える | ❌ Business以上 |
| Enterpriseは単独で使える | ❌ GitHub Enterprise Cloud必須 |
| パブリックコードフィルターはPro以上 | ❌ 全プランで利用可能 |
