## スライド4: 講座概要

この講座は、GitHub Copilot認定資格の合格を目指す方のための総合対策コースです。

この講座は、試験ドメインの配点比率を考慮した構成になっています。
各セクションには確認テストがあり、最後には模擬試験で理解度を測定できるようになっています。
確認テストや模擬試験で分からなかった、間違えた問題については、レクチャーに戻って復習すると効果的です。

また、ただ試験に合格するだけでは意味がありません。皆さんの業務ですぐに生かせるにように実践的なハンズオンも含んでいます。

---

## スライド5: 想定受講者

この講座がどのような方を対象としているかを説明します。

GitHub Copilotを使った開発を学びたい方、GitHub Copilot認定資格の取得を目指す方、AI支援開発やAIペアプログラミングに興味がある開発者の方が対象です。また、組織でのGitHub Copilot導入を検討している方や、プロンプトエンジニアリングを学びたい方にも適しています。

もし、皆様の期待に沿える内容でなかった場合は、Udemyの返金ポリシーに基づき、購入後30日以内であれば返金手続きが可能ですので、ご安心ください。

---

## スライド6: 講座の注意事項

講座を受講する前に、一点お伝えしたいことがあります。

本講座の座学編は、AIで音声を作成しております。受講者の皆様に聞き取りやすく、一貫した声をお届けするためにAI音声を活用しています。ただし、内容と説明については、講師が厳密に作成しております。

もし受講時にAI音声が好まれない場合は、購入後でもUdemy上で返金手続きが可能ですので、ご安心ください。

---

## スライド7: 本講座の進め方

続いて、この講座の進め方を説明します。

本講座には四つの学習要素があります。
一つ目は座学編です。機能の解説や概念の説明を行います。
二つ目はハンズオン編です。VS Codeで実際に操作しながら学びます。
三つ目は確認テストです。各セクションの末で理解度をチェックします。
四つ目は模擬試験です。最終セクションで本番に向けた実力測定を行います。模擬試験は2回あり、合計60問で構成されています。

座学で知識を身につけ、ハンズオンで実践し、テストで理解度を確認するというサイクルで学習を進めてください。

## スライド10: GitHub Copilot試験の概要と出題範囲

## スライド11: GitHub Copilot試験とは

GitHub Copilot試験は、GitHub Copilotに関する公式認定資格試験です。

試験の基本情報をお伝えします。試験名はGH300です。
レベルはIntermediate、つまり中級レベルです。
対象者は開発者、管理者、プロジェクトマネージャーとなっています。
認定の有効期間は2年間です。
提供元はGitHubで、Microsoftと連携しています。

---

## スライド12: 試験の対象者

この試験がどのような方を対象としているかを詳しく見ていきましょう。

一つ目はソフトウェア開発者です。GitHubを使用したコーディングに精通している方が対象です。二つ目は管理者です。GitHub Copilotの組織導入を担当する方が該当します。三つ目はプロジェクトマネージャーです。AI支援開発のワークフロー最適化を検討する方が対象となります。

試験のポイントとして覚えておいてほしいことがあります。この試験では、GitHubの基本知識とCopilotの実践経験が求められます。

---

## スライド13: 試験ドメインと配点比率

試験は7つのドメインで構成されています。配点比率と合わせて確認しましょう。

Domain 1の責任あるAIは7%です。Domain 2のGitHub Copilotのプランと機能は31%です。Domain 3のGitHub Copilotのしくみとデータ処理は15%です。Domain 4のプロンプト作成とプロンプトエンジニアリングは9%です。Domain 5のAIの開発者ユースケースは14%です。Domain 6のGitHub Copilotを使用したテストは9%です。Domain 7のプライバシーとコンテキスト除外は15%です。

---

## スライド14: 重点学習ドメイン

試験のポイントとして、配点比率の高いドメインを重点的に学習しましょう。

最も重要なのはDomain 2のGitHub Copilotのプランと機能です。これは31%と最大の配点を占めています。

次に重要なのは15%を占める二つのドメインです。Domain 3のしくみとデータ処理、そしてDomain 7のプライバシーとコンテキスト除外です。

これらのドメインは試験全体の約6割を占めていますので、しっかり学習してください。

## スライド15: GitHub Copilot試験の概要と出題範囲

続いて、各ドメインの詳しい出題内容を見ていきましょう。

---

## スライド16: 責任あるAI（7%）

まずはDomain 1の責任あるAIについてです。

このドメインでは、AIの使用に関連するリスクと制限事項について問われます。生成AIツールの限界、例えばソースデータやバイアスなどについても出題されます。また、AI出力の検証の必要性も重要なトピックです。

さらに、生成AIの潜在的な害についても理解が必要です。バイアス、セキュリティ、公平性、プライバシー、透明性といった観点があります。潜在的な害の軽減方法や、倫理的AIの説明についても学習しておきましょう。

---

## スライド17: GitHub Copilotのプランと機能（31%）

続いて、Domain 2のプランと機能です。31%と最大の配点を占める重要なドメインです。

出題内容としては、Copilot Individual、Business、Enterpriseの違いがあります。IDEでのGitHub CopilotとCopilot Chatの使い方も問われます。

Copilotのトリガー方法、つまりチャット、インラインチャット、提案、CLIなどについても理解が必要です。各プランの主要機能、スラッシュコマンドの使い方も重要です。

EnterpriseプランではKnowledge Baseの設定と活用についても出題されます。CLIでのCopilot使用方法も含まれます。

---

## スライド18: GitHub Copilotのしくみとデータ処理（15%）

Domain 3はしくみとデータ処理です。15%の配点があります。

このドメインでは、コード提案のデータパイプラインライフサイクルについて問われます。コンテキスト収集とプロンプト構築のプロセスも重要です。

プロキシサービスとフィルター処理についても理解が必要です。LLM、つまり大規模言語モデルのレスポンス生成やデータフローについても出題されます。

また、LLMの制限事項、例えばコンテキストウィンドウやデータの古さなどについても学習しておきましょう。

---

## スライド19: プロンプトエンジニアリング（9%）

Domain 4はプロンプトエンジニアリングです。

出題内容としては、プロンプトのコンテキスト決定方法があります。プロンプトの構成要素と言語オプションについても問われます。

ゼロショットとフューショットプロンプティングの違いも重要なトピックです。チャット履歴の活用方法、プロンプト作成のベストプラクティスについても理解が必要です。

プロンプトエンジニアリングの原則とプロセスフローについても学習しておきましょう。

---

## スライド20: AIの開発者ユースケース（14%）

Domain 5は開発者ユースケースです。14%の配点があります。

AIによる開発者生産性向上のユースケースについて問われます。新しいプログラミング言語やフレームワークの学習支援も含まれます。

ドキュメント作成やサンプルデータ生成といった活用方法も出題されます。レガシーアプリケーションの近代化、デバッグ、リファクタリングについても理解が必要です。

SDLC、つまりソフトウェア開発ライフサイクルにおける活用や、Productivity APIの使用方法についても学習しておきましょう。

---

## スライド21: テスト（9%）

Domain 6はテストです。

このドメインでは、ユニットテストと統合テストの生成について問われます。エッジケースの特定とテスト提案も重要なトピックです。

既存テストの効果向上についても理解が必要です。ボイラープレートコードの生成、アサーション作成の支援、テストデータの生成についても出題されます。

---

## スライド22: プライバシーとコンテキスト除外（15%）

最後に、Domain 7のプライバシーとコンテキスト除外です。15%の配点があります。

出題内容としては、SKU別のプライバシー考慮事項があります。コンテンツ除外の設定と効果についても問われます。

Copilot出力の所有権についても理解が必要です。重複検出フィルターと契約上の保護についても学習しておきましょう。

GitHub.comでのCopilot設定や、トラブルシューティング、例えば提案が表示されない場合の対処法なども出題されます。

以上で、セクション1のコース紹介と試験概要の説明は終了です。次のセクションからは、各ドメインの詳細な内容について学習していきましょう。
